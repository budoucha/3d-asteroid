# 奥行き深度スケール（3D ASTEROID）

アナグリフ描画の各要素がどの奥行きに存在するかを、正規化した深度 **D** で整理する。

## 定義

- **D = 0**: UI表示面（視差ゼロ = 画面面）。ワールド座標 `z = Z0 = 1000`。
- **D = 100**: グリッド最奥（縦線終端）。ワールド座標 `z = ZFAR = 9000`。
- 変換式: `D = (z - 1000) / 80`（逆に `z = 1000 + 80 * D`）
- **D が負** = 画面面より手前（自機側へ飛び出す方向）

## 各表示の基準・限界

| 分類 | 要素 | ワールド z | 深度 D | 備考 |
|---|---|---|---|---|
| **基準面** | UI（HUD/PAUSE/GAME OVER/タイトル） | 1000 | **0** | 常に画面面。視差ゼロで固定 |
| | 自機の基準面（機体中心） | 1000 | **0** | 左右上下のみ移動、z は不動 |
| | 当たり判定面 | 1040 | **0.5** | 隕石がこの面通過時に判定 |
| **自機** | 機首（最奥） | 1230 | **2.9** | 画面奥へ向く姿勢 |
| | 翼端・尾部（機体の最手前） | ~972 | **-0.35** | わずかに画面より手前 |
| | エンジン炎の先端 | ~891 | **-1.4** | 視点側へ噴射（ゆらぎで変動） |
| | 軌跡（翼端→後方へ流れ消滅） | 977→680 | **-0.3 → -4.0** | D≈-4 で消える |
| **隕石** | 出現（最奥） | 9000 | **100** | ここから手前へ接近 |
| | 消滅（最手前） | 520 | **-6.0** | すり抜け後この深度で除去 |
| | クレーター（近距離 z≈1500時） | z_eff≈923 | **-1.0** | 自機より少し手前に浮く |
| | クレーター（極近 z≈1150時） | z_eff≈770 | **-2.9** | 片眼6px上限で頭打ち |
| | クレーター（遠距離 z≈6500時） | z_eff≈4953 | **49** | 岩面とほぼ一致（僅か手前） |
| **背景** | グリッド最奥（縦線終端） | 9000 | **100** | 基準（=D100） |
| | グリッド最手前 | 1150 | **1.9** | 通路の手前端 |
| | グリッド横線の実効最奥 | ~9550 | **~107** | スクロール周期でD100を僅かに超える |
| | 星: 初期分布 | 1000–9000 | **0–100** | 通路全域にばら撒き |
| | 星: リサイクル下限 | 700 | **-3.75** | これより手前で最奥（D100）へ戻す |

## 設計メモ

- プレイ領域の奥行きは **D -6 〜 100**。隕石はこの全域を D100→-6 で走破する。
- **D=0（画面面）が UI・自機・当たり判定の基準**。ここより奥（正）が「これから来る脅威」、手前（負）が「すり抜けて飛び出したもの」。
- 手前側（負）に出るのは意図的な演出: 隕石のすり抜け（-6）、エンジン炎（-1.4）、軌跡（-4）、近距離クレーター（-1〜-2.9）。UI だけは常に D=0 で、飛び出す物体と交差しても視差ゼロなので見やすさを保っている。
- クレーターは「遠いと岩面（D≈49@z6500）、近いと自機より手前（D-1〜-3）」と、接近に応じて立体感が増す作り。
- グリッドの「最奥」は構造的な終端（縦線 = ZFAR = 9000）を D100 の基準にしている。スクロールする横線だけは周期の都合で瞬間的に D≈107 まで顔を出す。

## 関連コード定数（index.html）

```js
const Z0   = 1000;   // 画面面（視差ゼロ） = D0
const ZFAR = 9000;   // 隕石の出現距離     = D100
const ZCUT = 520;    // 隕石が消滅する手前限界 = D-6.0
```
